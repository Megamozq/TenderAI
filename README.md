## Обзор

TenderGuard AI анализирует тендерные закупки и присваивает каждому тендеру **балл подозрительности от 0 до 100%** с помощью машинного обучения без учителя — размеченная база «коррупционных» тендеров не нужна.

Система обнаруживает 18 косвенных признаков коррупции («красных флагов») по 6 категориям: временны́е аномалии, сговор участников, ценовые манипуляции, подозрительная документация, нарушения у поставщика, исторические паттерны. Все сигналы объединяются в единый интерпретируемый балл с SHAP-объяснениями.

**Проблема, которую решает система:** Аналитики по антикоррупции обрабатывают тысячи тендеров в месяц. Большинство нарушений — картельные сговоры, подставные компании, заточенные ТЗ — невидимы для правил ручной проверки. TenderGuard помогает сосредоточиться на самых подозрительных лотах.

## Как это работает

```
Сырые данные тендера
        │
        ▼
Извлечение признаков ──── 18 числовых сигналов из каждого тендера
        │
        ▼
 ┌──────────────────────────────────────┐
 │          Двухуровневый скоринг        │
 │                                      │
 │  Isolation Forest ──► IF Score        │
 │  (обнаружение аномалий)              │
 │                                      │
 │  Групповые оценки ─► Взвешенная сумма │
 │  (6 тематических групп)              │
 └──────────────────────────────────────┘
        │
        ▼
Score Fusion: 55% Группы + 45% IF  →  final_score (0–100%)
        │
        ▼
SHAP объяснение  →  Топ-3 признака для каждого тендера
        │
        ▼
Уровень риска: LOW / MEDIUM / HIGH / CRITICAL
```

## Возможности системы

- **Без размеченных данных** — чистое обучение без учителя через Isolation Forest
- **18 признаков коррупции** по 6 категориям с настраиваемыми весами
- **SHAP объяснения** — каждое предсказание сопровождается топ-N ключевых признаков
- **Score Fusion** — объединяет экспертные правила с ML-оценкой аномальности
- **REST API** — FastAPI сервер со Swagger UI, пакетным скорингом и эндпоинтом explain
- **Интерактивное CLI демо** — проверить любой тендер за 30 секунд
- **60 автотестов** — полное покрытие: признаки, предсказание, сохранение/загрузка, граничные случаи
- **Пакетный скоринг** — до 1000 тендеров за один API-запрос
- **Полная интерпретируемость** — аналитик видит, какие именно сигналы сработали

## 18 признаков модели

### Группа 1 — Временны́е паттерны (вес: 25%)

| Признак | Подозрительно когда... |
|---------|------------------------|
| `minutes_before_deadline` | Заявка подана за последние несколько минут |
| `published_on_friday` | Опубликован в пятницу вечером или в праздник |
| `acceptance_days` | Срок приёма заявок всего 1–3 дня |
| `tz_change_hours_before` | Техзадание изменено прямо перед дедлайном |

### Группа 2 — Участники (вес: 20%)

| Признак | Подозрительно когда... |
|---------|------------------------|
| `ip_collision` | Два и более участника вышли с одного IP-адреса |
| `winner_win_rate` | Победитель всегда выигрывает именно у этого заказчика |
| `director_overlap` | Директора компаний-участников пересекаются |
| `participants_count` | Только 1–2 участника на крупный лот |

### Группа 3 — Цена (вес: 22%)

| Признак | Подозрительно когда... |
|---------|------------------------|
| `price_reduction_pct` | Победитель не снизил цену совсем (заявка = НМЦК) |
| `min_bid_gap_pct` | Все заявки подозрительно близки друг к другу |
| `price_vs_market_pct` | Цена контракта сильно превышает рыночный уровень |

### Группа 4 — Документация (вес: 13%)

| Признак | Подозрительно когда... |
|---------|------------------------|
| `unique_spec_score` | ТЗ содержит уникальные требования под конкретный бренд |
| `tz_change_hours_before` | Спецификация изменена прямо перед закрытием подачи |

### Группа 5 — Поставщик (вес: 10%)

| Признак | Подозрительно когда... |
|---------|------------------------|
| `supplier_age_days` | Компания зарегистрирована менее 6 месяцев назад |
| `revenue_vs_contract` | Выручка поставщика меньше суммы контракта |
| `address_change_days` | Юридический адрес сменён незадолго до подачи |

### Группа 6 — История (вес: 10%)

| Признак | Подозрительно когда... |
|---------|------------------------|
| `amendment_count` | Много доп.соглашений заключено после победы |
| `subcontract_affiliation` | Субподряд уходит аффилированным с заказчиком структурам |
| `historical_win_rate` | Поставщик выигрывает почти в каждом тендере |

## Уровни риска

| Балл | Уровень | Рекомендуемое действие |
|------|---------|------------------------|
| 0–24% | 🟢 **LOW** | Тревоги нет |
| 25–49% | 🟡 **MEDIUM** | Поставить на мониторинг |
| 50–74% | 🟠 **HIGH** | Ручная проверка аналитика |
| 75–100% | 🔴 **CRITICAL** | Передать на расследование |

## Архитектура модели

### Isolation Forest

Основной алгоритм обнаружения аномалий. Изолирует выбросы путём случайного разбиения пространства признаков — подозрительные тендеры изолируются за меньшее число шагов (короткая длина пути), чем нормальные. Выбран потому что:

- работает **без размеченных данных** — истинное обучение без учителя
- масштабируется линейно с размером датасета
- устойчив к шуму и нерелевантным признакам
- параметр `contamination` задаёт ожидаемую долю аномалий

### Score Fusion

Итоговый балл объединяет два дополняющих сигнала:

```
final_score = (0.55 × групповые_оценки + 0.45 × isolation_forest_score) × 100
```

Гибридный подход позволяет выявлять как **известные паттерны** (совпадение IP → правило срабатывает), так и **неизвестные схемы** (новая картельная договорённость → Isolation Forest находит аномалию).

### SHAP объяснения

Каждое предсказание объясняется через SHAP (SHapley Additive exPlanations) — теоретико-игровой подход, который справедливо распределяет вклад каждого из 18 признаков в итоговый балл. Аналитик видит, какие именно сигналы подняли или снизили оценку, и с каким весом.

## Дисклеймер

TenderGuard AI — это **инструмент поддержки принятия решений**, а не юридический орган. Высокий балл риска указывает на статистические аномалии, заслуживающие проверки — это не доказательство нарушения. Все итоговые решения о расследовании принимаются квалифицированными аналитиками-людьми.
